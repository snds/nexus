# Color

Color in Nexus does two jobs: it builds the **neutral UI** (surfaces, borders, text, one accent)
and it carries **two orthogonal data channels** — entity *type* and *verdict* severity. Keeping
those channels separate is what lets a dense graph stay legible.

## The neutral system

A single neutral ramp drives every surface and text level; one accent ramp drives interaction.
The semantic aliases are the contract:

Full ramps live in **[Tokens → Primitives](/docs/tokens/primitives)**; the aliases in
**[Tokens → Semantics](/docs/tokens/semantics)**.

## Two systems for color-as-identifier

Color carries two kinds of meaning that follow **opposite encoding rules**, so the system keeps
them in **disjoint hue zones** that never cross-talk.

| System | Data type | Rule | Hue zone |
|---|---|---|---|
| **Identity** — entity *type* | Nominal (no order) | Hue only; perceptually distinct; grouped into families; **+ shape + icon** | purples · teals · metals (the **non-reserved** hues) |
| **Intent** — verdict severity, status/alerts | Ordinal + semantic | A **diverging ramp** (worst→best) + conventional status; **+ icon + label** | **reserved**: red · orange · amber · grass · blue · gray |

The reason this matters: hue is a **nominal-only** channel (it can say "different," never
"worse"). So entity type → hue is right, but verdict severity → a *diverging ramp* (read by the
warm→cool progression), not a hue rainbow. And because the warm "danger" hues are **reserved** for
intent, an entity's color can never be red/amber/green — so a node's color can't be mistaken for a
danger signal. Identity skews cool by design, which makes the warm severity **pips** pop.

## Channel 1 — entity type (identity zone)

A node's color answers "what kind of thing is this?" — never how dangerous. Categorical color is
reliable to ~8 hues, so color encodes **8 super-category families** (each a maximally-distinct hue
≥39° apart across the full wheel); **shape + icon** read the specific type within a family.

## Channel 2 — verdict severity (intent zone)

Verdict is an **ordinal diverging ramp** on the shared [intent axis](/docs/tokens/semantics),
shown via the [VerdictPip](/docs/components/data-display/verdict-pip) /
[VerdictBadge](/docs/components/data-display/verdict-badge) — never borrowing entity hues.

## Rules

- **Reserve the intent hues.** Red/orange/amber/grass/blue/gray mean *outcome* (status, severity) —
  never identity, never decoration. Entity color comes only from the identity zone.
- **Don't cross the channels.** A malicious actor is a *purple* (actor) node with a *red* (malicious)
  pip — never a red node.
- **Color is redundant, never sole.** Type also carries shape + icon + label; status also carries
  icon + label. This is what licenses a 14-type palette and keeps it colorblind-safe —
  see [Accessibility](/docs/foundations/accessibility).
- **Two background contexts.** The canvas (`--nx-bg`) sits a step below the chrome so the graph
  reads as its own plane.
