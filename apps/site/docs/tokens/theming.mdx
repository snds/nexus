# Theming & modes

The system is **dark-first**, and the theme switch lives in **one layer only — the semantics**.
Primitives are flat constants exposing both light and dark Radix values side by side; `:root`
points the semantics at the dark set and `:root.light` re-points the *same* semantics at the light
set. Primitives themselves never change between themes.

## How a mode switch works

```css
:root {                      /* dark — the default: semantics → dark primitives */
  --nx-bg: var(--nx-neutral-dark-1);
  --nx-fg: var(--nx-neutral-dark-12);
  --nx-accent: var(--nx-accent-dark-9);
}
:root.light {                /* light — same semantics → light primitives */
  --nx-bg: var(--nx-neutral-light-1);
  --nx-fg: var(--nx-neutral-light-12);
  --nx-accent: var(--nx-accent-light-9);
}
```

Components read semantics (`var(--nx-bg)`) and never change — they follow whichever primitive set
the active theme selected. This is the payoff of the [tier model](/docs/tokens/token-model):
theming is a one-layer edit, and primitives stay single-mode.

## Color values & opacity

Colors are real **Radix Colors** values (hex), consumed directly as `var(--token)` — never wrapped
in `hsl()`. Opacity comes from Tailwind opacity utilities or `color-mix()` — e.g. the translucent
node fill is `color-mix(in srgb, var(--nx-bg) 48%, transparent)` — never an inline alpha channel.

## Contrast across modes

Light mode was validated to keep the canvas legible (the demo canvas binds to `--nx-bg`, which
flips with the theme — 16.8:1, not the pinned-black it once was). See
**[Foundations → Accessibility](/docs/foundations/accessibility)**.
