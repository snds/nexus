## Two color systems, two rule sets

Color-as-identifier splits into two orthogonal systems drawn from **disjoint hue zones**, so they
never cross-talk (see [Foundations → Color](/docs/foundations/color)):

- **Intent / outcome** (ordinal + semantic) — verdict severity and status alerts both alias one
  shared ramp. Reserved hues: red · orange · amber · grass · blue · gray. Always with an icon + label.
- **Identity** (nominal) — entity *type*, drawn only from the **non-reserved** hues (purples,
  teals, metals…), grouped into families, paired with shape + icon. A type color **never** uses a
  reserved intent hue, so a node's color can't falsely signal danger.

Each channel maps to a full Radix **hue stack**: the token resolves to step 9, and bg (step 3),
border (6), and text (11) of that hue are reachable via `--nx-<hue>-<light|dark>-<step>`.

## Intent / outcome (`--intent-*`) — the shared axis

<SwatchGrid tokens={[['--intent-danger','danger → red'],['--intent-caution','caution → orange'],['--intent-warning','warning → amber'],['--intent-success','success → grass'],['--intent-info','info → blue'],['--intent-neutral','neutral → gray']]} />

Status alerts alias it directly: `--alert-error → danger` · `--alert-warning → warning` ·
`--alert-success → success` · `--alert-info → info`.

## Severity (verdict) — a diverging ramp on the intent axis

Ordinal, worst → best, read by the warm→cool progression (not a rainbow):

## Entity (color = type) — identity zone only

Nominal categories, grouped into hue families (adversary · artifact · web · host · identifier ·
file · comms · meta), disambiguated within a family by shape + icon:

## Radius & motion

- **Radius** — `--nx-radius-sm` · `--nx-radius` · `--nx-radius-lg`.
- **Motion** — `--nx-motion-fast | base | slow` + `--nx-ease-out | standard`. See
  **[Foundations → Motion](/docs/foundations/motion)**.
