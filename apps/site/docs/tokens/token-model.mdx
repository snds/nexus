# Token model

Tokens are the single source of truth for appearance. Every value an app or component renders —
a color, a type size, a radius, a duration — traces back to a token. **No component hardcodes a
hue, size, or duration.** They live in `@nexus/tokens` (`tokens.css`) and are consumed directly as
CSS custom properties (`var(--nx-...)`). Color primitives are real **[Radix Colors](https://www.radix-ui.com/colors)**
scales — **Slate** (neutral), **Indigo** (accent), **Red** (destructive) — not hand-rolled ramps.

## The Radix 12-step contract

Each step of a Radix scale has one job. Crossing these contexts is the most common color bug, so
the system holds to them strictly — and a CI test enforces the contrast they guarantee.

| Steps | Role | Used for |
|---|---|---|
| **1–2** | App / subtle background | `--nx-bg`, `--nx-surface-1` |
| **3–5** | Component states | `--nx-surface-2/3` (raised, hover, active) |
| **6–8** | Borders | `--nx-border` (6), `--nx-border-strong` (7), `--nx-ring` (**8**) |
| **9–10** | Solid fills | `--nx-accent` (9), `--nx-accent-hover` (10) |
| **11** | Low-contrast text (AA) | `--nx-fg-muted`, `--nx-fg-subtle`, `--nx-accent-text` |
| **12** | High-contrast text | `--nx-fg` |

**Text comes only from 11/12; solids only from 9/10.** Accent-as-fill (`--nx-accent`, step 9) is a
*different token* from accent-as-text (`--nx-accent-text`, step 11) — step 9 is too light for text.

## Three tiers

```
L0  Primitives   Radix scales — --nx-neutral-{light,dark}-1…12, --nx-accent-…, the type scale
      ↓ aliased by
L1  Semantics    purpose names — --nx-bg, --nx-surface-2, --nx-border, --nx-accent, --nx-fg
      ↓ consumed by
L3  Component     scoped values composed from semantics (e.g. --nx-node-fill, --nx-node-blur)
```

| Tier | Example | Rule |
|---|---|---|
| **L0 Primitive** | `--nx-neutral-dark-1`, `--nx-text-lg`, `--nx-accent-dark-9` | Radix scales, flat **single-mode constants** (light + dark side by side). Never referenced directly by a component. |
| **L1 Semantic** | `--nx-bg`, `--nx-fg-muted`, `--nx-accent`, `--nx-ring` | Purpose aliases that point at primitives. This is what components consume — and the **only layer with modes**. |
| **L3 Component** | `--nx-node-fill`, `--nx-node-blur` | Component-scoped values composed from semantics. |

The discipline of the tiers is what makes theming a one-layer change: primitives never change,
and **only the L1 semantics re-point** between the light and dark primitives — every component
follows automatically. Opacity uses Tailwind utilities or `color-mix()`, never inline hsl alpha.

## Naming

- `--nx-*` — the design system's own properties (the source of truth).
- `--entity-*` — the entity-type color channel (actor, malware, domain…).
- `--severity-*` — the verdict channel (malicious, suspicious, benign…).
- A **shadcn bridge** (`shadcn-bridge.css`) maps shadcn names (`--background`, `--popover`,
  `--ring`, `--radius`) onto the `--nx-*` semantics, so the L1 shadcn/Radix bases theme for free.
