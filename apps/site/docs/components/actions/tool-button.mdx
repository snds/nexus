# ToolButton

## Usage

Use `ToolButton` for compact, repeated icon actions in a toolbar — zoom, fit, save, highlight
path. The `label` is **required**: it both names the action for screen readers and renders as the
tooltip. Set `active` for a toggle that's currently on.

**When not to use:** for a labelled action, use **[Button](/docs/components/actions/button)**.

## Variants & states

<Specimen>
  <ToolButton icon="add" label="Zoom in" />
  <ToolButton icon="remove" label="Zoom out" />
  <ToolButton icon="fit_screen" label="Fit to screen" />
  <ToolButton icon="route" label="Highlight path" active />
  <ToolButton icon="save" label="Save" filled />
</Specimen>

`active` applies the selected treatment; `filled` flips the icon's FILL axis (reserve for active /
emphasis, per the [fill principle](/docs/foundations/principles)).

## Accessibility

- The `label` is applied as the accessible name **and** the tooltip content — never ship an
  unlabelled icon button.
- Tooltip is the Radix base: opens on hover **and** keyboard focus, dismisses on Escape, portalled.
- `tooltipSide` positions the tooltip away from screen edges (e.g. `left` for a right-rail toolbar).
