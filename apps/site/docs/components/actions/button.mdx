## Usage

Use a button to trigger an action — submit, save, open a dialog. Use exactly **one** primary
(`default`) button per view to keep the action hierarchy legible; everything else is `secondary`,
`outline`, or `ghost`.

**When not to use:** for navigation between pages, use a link. For a square icon-only control in a
toolbar, use **[ToolButton](/docs/components/actions/tool-button)**.

Hover, focus-visible, and active states are token-driven and identical to the shadcn base — real
keyboard focus rings via `--nx-ring`.

## Accessibility

- Renders a native `<button>`; fully keyboard operable (Enter / Space).
- `focus-visible` ring uses `--nx-ring` (accent-9) for AA contrast against every surface.
- Pass `asChild` to render as a link while keeping button semantics where appropriate.
- Disabled buttons are removed from the tab order — pair with a tooltip if the reason isn't obvious.
