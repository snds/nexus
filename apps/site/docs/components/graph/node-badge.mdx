## Usage

`NodeBadge` is the single node component for the entire canvas — the demo graph renders it through
a thin `Entity → NodeBadge` adapter. It encodes the three orthogonal channels and folds VIP /
status / imposter / +N indicators into one top-right cluster. Every node shares a translucent
token fill behind a backdrop blur so edges recede behind it.

**When not to use:** outside a graph context, prefer a labelled
**[VerdictBadge](/docs/components/data-display/verdict-badge)** + **[EntityIcon](/docs/components/iconography/entity-icon)**.

## Accessibility

- Verdict, type and family are three redundant channels — never color-only.
- Each indicator in the cluster carries a tooltip naming it (VIP, impacted, imposter, +N hidden).
- The `hub` role enlarges the glyph and flips FILL for the root, per the fill principle.
