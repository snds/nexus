## Usage

Use a state block whenever a region has no data, failed to load, or is loading. It gives every
such moment the same icon + title + message + optional action, so the product never shows a blank
or an ad-hoc error.

**When not to use:** for a transient confirmation, use a toast/notification. For a blocking
decision, use a **[Modal](/docs/components/overlays/modal)**.

## Accessibility

- The title is a heading; the message is descriptive text — screen readers get the full context.
- Loading states should also expose a live-region status where the wait is non-trivial.
- The optional `action` is a real focusable control.
