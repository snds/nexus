## Usage

Use a tooltip to name an unlabelled control or add a brief clarification. Keep content to a few
words — it is **supplementary**, never the only place critical information lives.

**When not to use:** don't put interactive content (links, buttons) inside a tooltip — use a
**[Popover](/docs/components/overlays/popover)**. Don't use it for long help text.

## Example

<Specimen>
  <Tooltip label="A real DOM tooltip" side="top">
    <button style={{ height: 36, width: 36, borderRadius: 6, border: '1px solid var(--nx-border)', background: 'var(--nx-surface-2)', color: 'var(--nx-fg)' }}>★</button>
  </Tooltip>
  <Tooltip label="Opens on focus too" side="right">
    <button style={{ height: 36, width: 36, borderRadius: 6, border: '1px solid var(--nx-border)', background: 'var(--nx-surface-2)', color: 'var(--nx-fg)' }}>?</button>
  </Tooltip>
</Specimen>

Hover or **keyboard-focus** the trigger. A single `TooltipProvider` (200 ms delay) is shared so
moving between controls doesn't re-wait.

## Accessibility

- Opens on hover **and** focus; dismisses on Escape and blur.
- Content is portalled and wired to the trigger — no manual ARIA needed.
- The trigger must be focusable (a real `<button>` / control). Don't wrap plain text.
