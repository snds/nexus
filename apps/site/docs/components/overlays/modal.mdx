## Usage

Use a modal for a focused, interrupting task the user must complete or dismiss — confirming a
destructive action, saving a graph, an auth step. Keep it to a single job; multi-step flows use a
**[Stepper](/docs/components/navigation/stepper)** inside.

**When not to use:** for non-blocking, contextual content anchored to a control, use a
**[Popover](/docs/components/overlays/popover)**. For passive notices, use an inline
**[StateBlock](/docs/components/feedback/state-block)**.

## Anatomy

A `Modal` renders a dimmed overlay + a centered surface containing a (visually-hidden or visible)
title, body content, and actions. It composes Radix `Dialog`, so focus trap, scroll lock, Escape,
and `aria-modal` come for free.

```tsx
import { Modal } from "@nexus/ui/nexus";

<Modal label="Save graph" open={open} onClose={() => setOpen(false)}>
  <SaveGraphForm />
</Modal>
```

## Accessibility

- Focus is trapped while open and restored to the trigger on close.
- Escape and overlay-click both call `onClose`.
- `label` becomes the dialog's accessible name (a `DialogTitle`, visually hidden if not rendered).
- Background content is inert (`aria-modal`).
