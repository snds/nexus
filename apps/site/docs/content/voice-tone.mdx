# Voice & tone

Nexus is a security analysis tool. Its voice is **calm, precise, and respectful of the analyst's
expertise.**

## Voice (constant)

- **Direct.** Lead with the fact. "3 malicious nodes" — not "We found that there may be some nodes
  that could be malicious."
- **Precise.** Use the exact term ([terminology](/docs/content/terminology)); don't soften or
  hedge security facts.
- **Confident, not alarmist.** Severity is carried by the verdict channel — copy doesn't need
  exclamation marks or red words.

## Tone (varies by context)

| Context | Tone | Example |
|---|---|---|
| Verdict / result | Neutral, factual | "Verdict: malicious." |
| Empty state | Helpful, light | "Saved graphs will appear here." |
| Error | Plain, accountable | "Couldn't load the graph. Try again." |
| Destructive confirm | Serious, specific | "Delete this graph? This can't be undone." |
| Success | Brief | "Graph saved." |

## Don't

- Don't blame the user ("You entered an invalid query") — describe the state ("No results for that
  query").
- Don't dramatize. The data is serious enough; the copy stays even.
- Don't pad. Every word earns its place in a dense UI.
